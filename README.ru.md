# TG Post Search

[English version](README.md)

Telegram Post Search & Monitoring — инструмент для сбора, поиска и мониторинга сообщений из Telegram-каналов с веб-интерфейсом.

## Возможности

- Пул Telegram-аккаунтов с ротацией при flood-wait
- Инкрементальный сбор сообщений из каналов
- Мониторинг по ключевым словам с уведомлениями в Telegram (текст и regex)
- Полнотекстовый поиск: локальная БД, напрямую в Telegram, AI-поиск (LLM)
- Веб-панель (FastAPI + Pico CSS)
- HTTP Basic Auth
- Готов к запуску в Docker

## Быстрый старт

### Требования

- Python 3.11+
- API-ключи Telegram с [my.telegram.org/apps](https://my.telegram.org/apps)

### Установка

```bash
pip install .
```

Скопируйте и заполните переменные окружения:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```
TG_API_ID=ваш_api_id
TG_API_HASH=ваш_api_hash
WEB_PASS=ваш_пароль
LLM_API_KEY=               # опционально, для AI-поиска
```

При необходимости отредактируйте `config.yaml` (значения по умолчанию работают из коробки).

Запустите сервер:

```bash
python -m src.main serve
```

Откройте http://localhost:8080 в браузере.

### Docker

```bash
cp .env.example .env
# заполните .env своими данными
docker-compose up -d
```

## Конфигурация

### Переменные окружения (.env)

| Переменная | Обязательна | Описание |
|---|---|---|
| `TG_API_ID` | Да | Telegram API ID |
| `TG_API_HASH` | Да | Telegram API Hash |
| `WEB_PASS` | Да | Пароль веб-панели |
| `LLM_API_KEY` | Нет | API-ключ для AI-поиска |

### config.yaml

Конфиг поддерживает подстановку `${ENV_VAR}`. Пустые переменные окружения игнорируются (применяются значения по умолчанию).

| Секция | Описание |
|---|---|
| `telegram` | API-ключи (`api_id`, `api_hash`) |
| `web` | Хост, порт, пароль (по умолчанию: `0.0.0.0:8080`) |
| `scheduler` | Интервал сбора, задержки, лимиты, макс. flood wait |
| `notifications` | `admin_chat_id` для уведомлений о совпадении ключевых слов |
| `database` | Путь к SQLite (по умолчанию: `data/tg_search.db`) |
| `llm` | Провайдер LLM, модель, API-ключ, флаг включения |

## Использование

### CLI-команды

```bash
# Запуск веб-сервера
python -m src.main [--config CONFIG] serve [--web-pass PASS]

# Разовый сбор сообщений
python -m src.main [--config CONFIG] collect

# Поиск по локальной базе
python -m src.main [--config CONFIG] search "запрос" [--limit N]
```

### Веб-интерфейс

| Страница | Путь | Описание |
|---|---|---|
| Дашборд | `/` | Статистика, статус планировщика, подключённые аккаунты |
| Авторизация | `/auth/login` | Добавление Telegram-аккаунтов (телефон + код + 2FA) |
| Аккаунты | `/accounts` | Управление аккаунтами (вкл/выкл, удаление) |
| Каналы | `/channels` | Добавление/удаление каналов, управление ключевыми словами |
| Поиск | `/search` | Поиск сообщений (локальный / Telegram / AI) |
| Планировщик | `/scheduler` | Запуск/остановка/ручной запуск автоматического сбора |

### Порядок работы

1. Запустите сервер (`python -m src.main serve`)
2. Откройте веб-панель и добавьте Telegram-аккаунт через страницу авторизации
3. Добавьте каналы для мониторинга
4. (Опционально) Настройте ключевые слова для уведомлений
5. Запустите планировщик или выполните ручной сбор
6. Ищите по собранным сообщениям
